# 妈祖祈福系统 - 移动端开发文档

## 1. 项目概述
本项目是一款基于纯前端技术栈构建的 H5 移动端互动祈福应用。页面风格采用 **Q版可爱中国风**，优化了原型图中较为生硬的视觉逻辑。

### 核心功能
* **启动动画**：加载载入动效。
* **主页（祈福大厅）**：
  * **祈福烧香互动**：支持自定义/预置祈福语，提供插香、点燃、香烟缭绕动画；妈祖神像会有睁眼、微笑反馈，并附带祝福语弹窗。
  * **掷杯（圣杯问卜）**：支持自定义/预置祈福语，执行 3 轮掷杯判定（成功率设为 80%）。每次掷杯通过 3D 旋转展示阴阳杯的不同组合（圣杯、阳杯、阴杯）。若连续 3 次圣杯则获得大成祝福（妈祖睁眼微笑+喜庆动效）；中途失败则展示鼓励动效（妈祖沮丧）。
* **个人中心（信息维护）**：预置用户数据，支持增删改查。
* **吉色功能（星盘推荐）**：剔除原界面的输入框，直接调用个人中心当前选中的用户信息，结合性别、生辰八字、紫微星象逻辑，动态运算并推荐今日吉色、开运穿搭与饰品。

---

## 2. 技术架构
* **展示层**：HTML5 + CSS3 (Tailwind CSS 3 动画、过渡、渐变支持)
* **逻辑控制**：Vue 3 (基于 CDN 引入，免去复杂的构建步骤，方便开箱即用)
* **矢量动画**：全部核心道具（妈祖神像、香炉、阴阳杯等）均采用 **纯 SVG 绘制及控制**，无需外部多余静态资源，保证流畅度。
* **本地存储**：`localStorage`（保存用户配置和祈福数据）

---

## 3. 核心设计与算法逻辑

### 3.1 掷杯（圣杯）判定算法
连续 3 次出现圣杯（一阴一阳）才算成功。
* 设定单轮总成功率为 $80\%$，故每次掷杯出现“圣杯”的概率 $P_{single} = \sqrt[3]{0.80} \approx 92.83\%$。
* 掷杯结果事件概率分配：
  * **圣杯 (阴+阳)**：$92.83\%$
  * **阳杯 (阳+阳)**：$3.585\%$
  * **阴杯 (阴+阴)**：$3.585\%$

### 3.2 吉色推荐算法 (基于生辰八字/紫微星象伪算法)
根据用户的出生日期（月、日）计算五行属性，并给出穿搭建议。
* 五行与颜色映射：
  * **木**：绿色、青色 (搭配：翡翠、木质手串)
  * **火**：红色、朱砂色 (搭配：朱砂手链、红围巾)
  * **土**：黄色、啡色 (搭配：蜜蜡、黄水晶)
  * **金**：白色、金色 (搭配：珍珠、银饰)
  * **水**：黑色、蓝色 (搭配：黑曜石、蓝宝石)

```javascript
    '.css': 'text/css',
    '.json': 'application/json',
    '.png': 'image/png',
    '.jpg': 'image/jpg',
    '.gif': 'image/gif',
    '.svg': 'image/svg+xml'
  };

  const contentType = mimeTypes[extname] || 'application/octet-stream';

  fs.readFile(filePath, (error, content) => {
    if (error) {
      if (error.code === 'ENOENT') {
        res.writeHead(404, { 'Content-Type': 'text/html' });
        res.end('<h1>404 Not Found</h1>', 'utf-8');
      } else {
        res.writeHead(500);
        res.end(`Server Error: ${error.code} ..\n`);
      }
    } else {
      res.writeHead(200, { 'Content-Type': contentType });
      res.end(content, 'utf-8');
    }
  });
}).listen(PORT, () => {
  console.log(`===========================================`);
  console.log(`  妈祖祈福阁 正在运行中...`);
  console.log(`  请在浏览器访问: http://localhost:${PORT}`);
  console.log(`===========================================`);
});
```

## 6. 运行与停止脚本

### 6.1 Windows 脚本

```cmd
@echo off

```

#### `stop.bat`
```cmd
@echo off
chcp 65001 >nul
echo [INFO] 正在搜寻端口 3000 上的服务器进程...
for /f "tokens=5" %%a in ('netstat -aon ^| findstr 3000') do (
    taskkill /F /PID %%a >nul 2>&1
    echo [SUCCESS] 已成功关闭 PID: %%a 的服务器进程。
    goto end
)
echo [WARN] 未找到运行在端口 3000 上的服务器进程。
:end
pause
```

### 6.2 Linux / macOS 脚本

#### `start.sh`
```bash
#!/bin/bash
echo "[INFO] 正在启动 妈祖祈福阁 本地服务器..."
node server.js &
SERVER_PID=$!
echo $SERVER_PID > server.pid

sleep 1.5

# 检测操作系统并自动唤起浏览器
if which xdg-open > /dev/null; then
  xdg-open http://localhost:3000
elif which open > /dev/null; then
  open http://localhost:3000
else
  echo "[SUCCESS] 服务已启动。请手动打开浏览器访问: http://localhost:3000"
fi
```

#### `stop.sh`
```bash
#!/bin/bash
if [ -f server.pid ]; then
  PID=$(cat server.pid)
  kill $PID 2>/dev/null
  rm server.pid
  echo "[SUCCESS] 已通过 PID 文件关闭 妈祖祈福阁 本地服务 (PID: $PID)。"
else
  echo "[INFO] 未检索到 PID 临时文件，正在尝试强制释放 3000 端口..."
  PID=$(lsof -t -i:3000)
  if [ -n "$PID" ]; then
    kill -9 $PID
    echo "[SUCCESS] 已关闭占用 3000 端口的进程 (PID: $PID)。"
  else
    echo "[WARN] 端口 3000 未被占用，无正在运行的服务。"
  fi
fi
```
